    Basic
    Not everyone wants to be a hardcore developer. Writing truly engaging content, that both delights and educates people, is an underrated talent and you shouldn't be required to learn all the intricacies of modern web frameworks just to produce awesome pages. In the past, the best you could have done would be to write content in your editor of choice, export it into markdown and then hope that some developer somewhere can incorporate it and add the styles, buttons, images and other visual features which are required to make content transformational, rather than just more boring information.
    This site changes all that.

    What is Mdx anyway?

    In short: it's your normal markdown - 
    md
    - with an added little 
    x
    factor. In practice, the 
    x
    means that we can 
    extend
    the capabilities of markdown and, in fact, create entire React components directly in our content files. If you don't know what a React component is, or why you would want to have them in your content files, don't stress! This section is all about showing you how this is actually a writing superpower, and how you can put it to best use when contributing.
    You might have used a markdown cheatsheet before to figure out how to handle headings, links, images, and other basic necessities. The beauty of 
    mdx
    is that all that stuff still works exactly the same. You'll still be writing what appears - for the most part - to be normal markdown. It's just that we can now do so much more...

    Images

    Let's take images, for example. If you ever written markdown content before, you'll be familiar with the way images are added, which remains the same with 
    mdx
    :
    basic-image.mdx
    ![Basic image example](../../../images/starfox.png)
    However, because we're extending markdown's capabilities, you could also choose to write:
    react-image.mdx
    <Image src="../../../images/starfox.png"/>
    In this case, the result is the same (except for the bottom margin, which we'll fix later). Using the traditional markdown format defaults to how Gatsby generally processes images, though learning how to use the 
    Image
    tag will allow you to start doing more complicated things later on, like specifying your own styles or passing specific properties to components which come ready-made with many different options.

    Links

    The same idea goes for links. You're likely used to writing markdown links like this:
    Awesome video
    basic-link.md
    [Awesome video](https://www.youtube.com/watch?v=Qzm_1Tuor1A)
    Which will still work exactly as you expect it to in 
    mdx
    files. However, sometimes a link just isn't enough. Sometimes, we want to make things a bit more fancy, which is not possible using plain old markdown. Happily, this as easy as specifying a proper React Image was:
    link to with an icon.
    react-link.mdx
    <Link to="https://www.chat.makerdao.com" icon="rocketchat">
  link to with an icon.
</Link>
    When writing relative links, be aware that you must write out the full route from after the locale, i.e. the Intermediate tutorial is linked to like this: 
    [Intermediate tutorial](/develop/content/intermediate)
    . If you want to link to other internal locales, then specify the locale as well, i.e. 
    es/develop/content/intermediate
    The Link component will check whether the link is internal/external; if the internal has a locale appened (
    fr/work_with_us/bounties
    ); and if external links have 
    https
    (http://google.com will automatically be transformed). Internal links are not directory based, like images, but route based as this is a more robust solution for a site with many different contributors.

    Sometimes, we'll need to embed a video which also links to a specific playlist. Inline videos do not allow us to do this, which is an issue for our Governance and Risk call series, amongst others. Therefore, if you're wanting to embed a video and link it within a specific YouTube playlist, we do have a bespoke 
    <Video />
    component for you. This serves as a helper for getting around youtube playlists, but we still recommend using the inline video spec (ie. video: https://urltoyoutubevideo.com) for regular inline videos.
    video-playlist.mdx

    Callouts and CTAs

    OK, so now we can handle images, links, buttons, and inline code - all of which will make our content really pop and we didn't have to ask a single developer to add anything to our work. What a relief... The last basic tool we want to introduce you to, before moving on to more detailed use of React components, is the Callout. Sometimes, you don't want the user to take action - you just need to draw attention to a very specific and important part of your content - a note or summary or key takeaway. We've got you covered.
    This is a callout with the (default) primary style and the default "warning" icon from dai-ui-icons!
    This is a callout with the (specified) secondary style and the "search' icon from dai-ui-icons!
    In addition to callouts, we can do Calls To Action. CTAs are really just good-looking green boxes to be used as shorthand when you want to draw attention to a particular piece of content. They do not change if you pass them any props. Callouts are fully fleshed out and they support icons. So, if you want colors and icons, use a Callout. If you just want a green box for attention, use a CTA. This can be achieved either with a 
    <CTA>
    React component tag, or the simply md blockquote syntax, i.e. 
    >
    .
    One important note is here is that you must ensure your content is on a separate line, with spaces in between. This is because of what we call the "Child Rule", which we explain in the next section.
    This is also a shorthand way of creating CTAs.
    callouts.mdx
    <Callout icon>

This is a callout with the (default) primary style and the default "warning" icon from dai-ui-icons!

</Callout>

<Callout icon="search" secondary>

This is a callout with the (specified) secondary style and the "search' icon from dai-ui-icons!

</Callout>

<CTA>

Important note!

</CTA>

    Status Banners

    At various stages in our docs, we need to be able to show the Status of a given project or process. This requires something slightly different from either a Callout or a CTA, so we have the 
    <StatusBanner>
    component which comes replete with flexible color props and a bunch of other cool features for the enterprising writer.
    This will stick with us until the end...
    status-banners.mdx
    <StatusBanner>

**You're becoming an mdx expert already! [Much wow](#)**

</StatusBanner>

<StatusBanner warning>

**You're becoming an mdx expert already! [Much wow](#)**

</StatusBanner>
    You're becoming an mdx expert already! Much wow
    You're becoming an mdx expert already! Much wow
    It gets even better though, because StatusBanner not only give you the ability to control colors and set your own should you require them. They can also receive the 
    sticky
    prop, which means that they will stick to the top of the screen as soon as the user scrolls past them, wherever they are in the page.
    sticky-status.mdx
    <StatusBanner sticky>

**This will stick with us until the end...**

</StatusBanner>
    And that's not all. You can even include a StatusBanner in your file's frontmatter, which means that it will appear above the breadcrumbs on your page. You can leverage the same options there, defining both its color and stickiness in the frontmatter itself. The one at the top of this page is defined below:
    frontmatter-status.mdx
    status: {text: "You'll learn how to create a banner like this in the tutorial below!", notice: true, sticky: false}
